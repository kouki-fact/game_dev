## 共同開発のルール（重要）
　
データが消えたり、自分の手元にだけファイルがない！という事態を防ぐためのルールです。

1. **作業開始時は必ず `git pull`**
   - 誰かが更新した内容を自分のPCに取り込んでから作業を始めます。
   - コマンド: `git pull origin main`

2. **`.meta` ファイルはセットで送る**
   - Unityの素材やスクリプトには、必ず同名の `.meta` が存在します。
   - `git add .` を使えば漏れがないので、必ず「点（.）」で追加しましょう。

3. **保存（コミット）の合言葉**
   - `[Add]`: 新機能・素材の追加
   - `[Fix]`: バグ修正
   - `[Update]`: 調整・更新

4. **シーン（Scene）の衝突回避**
   - 同じシーンファイルを同時にいじるとデータが壊れやすいため、触る前に必ず相談しましょう！

5. **変更したら必ずこのREADME.mdの記録各個人書く**
    - 最初の人は↓のようにテンプレートを書く

## 取り込みから保存して GitHub に送る手順
0. **git pull**
    - 他の人が追記した分を自分のPCに取り込む
1. **git add .**
   - 「どこが変わったか」をGitが自動スキャン
2. **git commit -m "○○を追加など"**  
    - この時、Gitは「前回のセーブポイントからの変化」だけを記録

3. **git push origin main**
    - GitHub側に「新しい変化分」だけ送り、GitHubがそれを自動で合流（マージ）させます。
